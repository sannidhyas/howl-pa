# Vault conventions

Howl PA writes into an existing Obsidian vault (default `~/Documents/projecthowl`). The bot inherits the vault's structure rather than imposing its own. If your vault uses different paths, export `VAULT_PATH` and update the constants in `src/vault.ts`.

## Expected folder layout

```
projecthowl/
├── 00_Dashboard/             # Home, ledger views (manual)
├── 01_System/                # Setup docs, consequences, privacy
├── 02_Plans/                 # Schedules, inbox, multi-year plans
│   └── inbox.md              # Bot appends recurring tasks here
├── 02_Templates/             # Daily, Literature, Monthly, Note
├── 03_Daily/                 # YYYY-MM-DD.md (bot-managed)
├── 04_Notes/
│   ├── inbox/                # Bot: classified "note" captures
│   └── 41_Literature/        # Bot: ZotLit-compatible stubs + summaries
│       └── 411_Attachments/
├── 05_Progress/              # Bot: weekly-review writes here
├── 06_Projects/              # Bot: /open promotes pipeline ideas here
│   ├── 61_…
│   └── 6N_PascalName/        # Next free slot, picked by src/idea-open.ts
├── 07_antithesis/            # Manual
├── 08_Pipeline/
│   └── ideas/                # Bot: new idea captures
│       └── YYYY-MM-DD-slug/
│           └── index.md
└── Attachments/
```

## Daily note frontmatter (`03_Daily/YYYY-MM-DD.md`)

Generated by `src/vault.ts:ensureDailyNote()` with weekday-aware defaults:

- `day_type`, `exempt`, `fasting` (Mon → `water_fast_until_14`, else `IF_08_14`)
- `gym_required` (Tue–Sat), `thesis_required` (Mon–Fri)
- `*_done` toggles the evening tracker flips
- Trackers: `artifact_type`, `sport`, `swim`, `sleep_hours`, `energy`, `soreness`, `kit_done`, `meditation_done`
- Penalty clearing fields from `01_System/Consequences.md`

Section layout the bot maintains:

- `## Brief` — written by `morning-brief` mission at 07:00
- `## Morning ritual` — written by the morning survey at 07:05
- `## Thesis artifact` — appended by `/thesis` captures
- `## Evening tracker` — written by the evening survey at 21:05

## Commit protocol

Every bot write is committed by `src/vault-writer.ts` with a `[mc] <op>: <path>` message so the obsidian-git plugin sees a clean tree. The bot never stages uncommitted user edits — it only stages what it wrote itself.

## What the bot will never do

- Create new top-level numbered folders (`0X_*`).
- Rename folders you already manage.
- Auto-delete notes. The only delete path is explicit: `/discard <slug>` for ideas in pipeline.
- Write into `Attachments/`, `07_antithesis/`, or any folder not listed above.
